Keep `README.md` up to date whenever making changes that affect documented behaviour.

**Why:** The user wants documentation to stay accurate as the project evolves.

**How to apply:**

- Update `README.md` when adding/changing LSP capabilities (completions, hover, diagnostics), supported commands, file associations, or the install/dev workflow.
- Commit the README together with the changed files in the same commit (or immediately after).
